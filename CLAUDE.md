# CLAUDE.md — память проекта

Контекст для ИИ-ассистента и разработчика. Держи актуальным.

## Что это

Сайт-поздравление с днём рождения для **Кристины**. Личный подарок.
Деплой: **GitHub Pages**, репозиторий `happy-birthday-kristina`.

## Сцена (одна страница)

1. **Распаковка подарка** — коробка с бантом; тап (или тряска телефона) → конфетти → сайт.
2. **Hero** — «С днём рождения, Кристина» + подзаголовок, оркестрованный staggered-вход.
3. **Финал** — одна большая буква **«Л»** — это личное «люблю». Тап по букве → салют из
   разных эмодзи-сердечек (`confetti.shapeFromText`, красных больше розовых), вылетающих
   из самой буквы; маленькая кнопка внизу **«Ещё конфетти»** — дополнительный залп.
4. **Ритм-игра** под «Л» — кнопка «Играть в такт» → жмёшь в такт уже играющей песне
   (одна полоса, одна кнопка = сама «Л»; стиль сердцебиения Dark Pictures). Проиграть
   нельзя: попадания = конфетти + комбо, промахи не карают; конец — на границе трека,
   музыка играет дальше. Разбор — `docs/DOCUMENTATION.md` §13.

> Ранее были тортик / галерея / письмо — **удалены намеренно** по просьбе владельца. Не возвращать.

## Стек и принципы

- Чистые **HTML + CSS + JS, без сборки и без фреймворка**. Открывается браузером напрямую.
- **Без рантайм-CDN для логики:** `canvas-confetti` вшит локально (`js/confetti.js`).
  Единственная внешняя загрузка — Google Fonts (Playfair Display + Manrope).
- **iPhone-first (Safari)** — главный приоритет качества.
- **PWA:** устанавливается на «Домой» (standalone) и работает офлайн (service worker кэширует
  app-shell + Google Fonts). Тоже статикой, без сборки — см. `manifest.json`, `sw.js`.

## Дизайн-направление

Одна тема — тёплый романтичный розовый. Принципы (по frontend-design):
доминирующий акцент `--accent` + золотой контрапункт `--gold`; атмосфера через
градиентный меш + тонкий SVG-шум + свечение за крупным текстом; связка шрифтов
**дисплейный курсив Playfair** (имя, «Л») + **Manrope** (текст, uppercase-трекинг в мелочах);
один срежиссированный вход со staggered-reveal вместо разрозненных микроанимаций.

Меняешь вид → правишь переменные в `:root` (`css/styles.css`). Больше тем нет — убраны намеренно.

## Гочи (не наступить повторно)

- **`hidden` + `display` в CSS:** любой явный `display` в авторских стилях перебивает
  HTML-атрибут `[hidden]`. Для скрываемых элементов держать правило `.x[hidden]{display:none}`
  (см. `.music-toggle`).
- **Автоплей аудио на iOS:** музыка стартует только внутри пользовательского жеста (тап по
  подарку/кнопке). Не пытаться автостартить без жеста.
- **`dvh`, а не `vh`:** иначе секции прыгают при скрытии адресной строки Safari.
- **`prefers-reduced-motion`:** уважать — частицы отключаются, reveal без сдвига.
- **Конфетти уважает reduced-motion** через `disableForReducedMotion: true`.
- **`background-clip: text` срезает свисающие части глифа:** завиток «Л» и italic-терминалы
  имени выходят за границы бокса, а фон красится только внутри бокса. Держать `padding` на
  `.finale-letter` и `.hero-name`, иначе чернила вне бокса не зальются (видно ровный срез).
- **Оптический центр «Л»:** глиф Playfair «Л» из-за правого бокового пробела смещён влево
  (~0.022em от центра advance-бокса). Выправлено `text-indent` на `.finale-letter`; при
  `text-align:center` indent двигает на половину, поэтому значение удвоено (~0.044em).
  **Меняешь шрифт/размер буквы → перепроверь оптический центр** (canvas `measureText`
  → `actualBoundingBoxLeft/Right`).
- **Зум на iOS отключён:** `viewport` c `user-scalable=no, maximum-scale=1` +
  `touch-action: manipulation` (снимает double-tap-зум). iOS игнорирует `user-scalable`,
  поэтому pinch дополнительно гасится в JS (`preventDefault` на `gesturestart/change/end`).
- **Overscroll-баунс:** фон-градиент продублирован на `html`, иначе по краям при «резинке»
  видны белые поля.
- **`background-attachment: fixed` — артефакты/швы на iOS Safari.** Фон-меш вынесен в отдельный
  фиксированный слой `<div class="bg">` (z-index −1). При этом `body` держать **прозрачным**
  (`background: transparent`), иначе его непрозрачная заливка перекроет `.bg` («градиенты исчезли»).
- **iOS не обрезает transform-промоутнутые дети фикс-слоя.** У `.floaties` (fixed) `overflow:hidden`
  на iOS НЕ обрезает частицы с `transform` → они вылезают за экран и дают горизонтальный скролл
  (в Chromium бага нет — потому превью его не показывало). Лечится `contain: layout paint size` на
  `.floaties` + держим частицы в границах по X (left 3–82vw, дрейф ±4vw в `main.js`).
- **Имя не влезает по ширине:** `fitName()` в `main.js` уменьшает кегль, если имя шире ~92%
  экрана (плюс размер уже на `clamp(...vw...)`). Меняешь имя в конфиге — оно само подгонится.
- **PWA-пути только относительные (`./`, `manifest.json`, `assets/...`):** сайт на GitHub Pages
  живёт в подпапке (`/happy-birthday-kristina/`). Абсолютные `/...` смотрят в корень домена и
  ломают офлайн/установку на Pages (локально с корня работали бы — легко не заметить).
- **iOS ≠ манифест:** установку на «Домой» на iPhone держат apple-мета
  (`apple-mobile-web-app-*`) + `apple-touch-icon`, не только `manifest.json`. Держать оба.
- **Версию кэша SW `birthday-vN` поднимать при смене закэшированных ассетов** — иначе у уже
  установивших останется старая версия (cache-first не идёт в сеть за свежим). SW не
  регистрируем на `file:` (иначе ошибка при прямом открытии файла).
- **Иконки PWA генерятся, не рисуются в рантайме:** `npm run icons` (`scripts/gen-icons.mjs`)
  рендерит ❤️ на розовом градиенте в PNG через реальный Chrome (`channel:"chrome"` —
  скачанного Chromium в проекте нет). PNG коммитятся. Путь-сердечко — симметричный Material
  «favorite» (самодельный несимметричный давал «кривое» сердце). **Меняешь тему/палитру →
  пере-генерируй иконки** (цвета зашиты в `gen-icons.mjs`, держать в синхроне с `:root`).
- **Ритм-игра: часы — только `audio.currentTime`.** Движок (`js/rhythm.js`) не заводит своих
  таймеров и не трогает аудио (не перематывает/не паузит) — иначе синк уедет. Игра стартует
  **с текущей позиции** уже играющего трека (не с нуля).
- **Конец игры — на границе лупа, не по своему таймеру.** `currentTime` у `<audio loop>`
  крутится `0…duration`; движок считает обороты (`nextLaps`, гистерезис 0.5 c). Конец =
  ближайшая граница `k·duration` с зазором ≥ `minPlay` (10 c) — старт под конец лупа не даёт
  мгновенную игру. Музыка после конца продолжает играть.
- **Карта битов — сгенерена, не ручная.** `js/rhythm-beatmap.js` собирается
  `tools/beatmap/extract.py` (librosa). **Меняешь песню → пере-собери карту** (см.
  `docs/DOCUMENTATION.md` §13). Текущий режим — `--accent` (ключевые онсеты по силе,
  неравномерный музыкальный ритм); `--top` — доля сильнейших, `--min-gap` — мин. пауза.
- **Полоса рисуется НАД буквой** (в разметке `#rhythm` до `#finale-letter`) — иначе рука
  при тапе по «Л» её закрывает. «Л» на попадание коротко «попает» (класс `.pop` +
  `@keyframes letter-pop`; специфичность правила выше, чем снятие idle-анимации).
- **Локальная разработка + service worker:** SW кэширует app-shell **cache-first**, поэтому
  при отладке браузер отдаёт СТАРЫЕ закэшированные html/js (правки на диске не видны).
  Лечится разово: снять регистрацию SW + очистить `caches` и перезагрузить (или поднять
  версию кэша). Проявляется как «мои правки не применились».
- **`targetPct` в `rhythm.js` синхронен с `.rhythm-target{left:…}` в CSS.** Кольцо-цель
  визуально должно стоять там, где движок засчитывает попадание. Меняешь одно — меняй другое.
- **Движок портируемый:** `rhythm.js` ничего не знает про день рождения (API через
  options/колбэки), данные песни — отдельно в `rhythm-beatmap.js`. Не вплетать сюда логику сайта.

## Запуск и проверка

```bash
python3 -m http.server 8000   # http://localhost:8000 (посмотреть глазами)
npm test                       # E2E: WebKit(iPhone)+Chrome(Android), ~25с
npm run icons                  # пере-генерировать PWA-иконки из ❤️ (разово, после смены темы)

# Пере-собрать карту битов ритм-игры (разово, при смене песни):
tools/beatmap/.venv/bin/python tools/beatmap/extract.py \
  assets/audio/<файл>.mp3 --src assets/audio/<файл>.mp3 \
  --out js/rhythm-beatmap.js --accent --min-gap 0.7 --top 0.5
```

**Тесты (Playwright) — основной способ верификации, быстрее ручного превью.**
`tests/e2e.spec.js` гоняет сайт в **реальном движке WebKit** (эмуляция iPhone, dpr 3) и
Chromium. Именно WebKit ловит iOS-специфику (горизонтальный скролл, швы фикс-слоёв),
которую Chromium-превью НЕ показывает — на этом мы не раз обжигались. Покрыто: нет
горизонтального переполнения, имя влезает и центрировано, «Л» оптически по центру,
частицы в границах вьюпорта, тап открывает только подарок, нет ошибок консоли, нет
выделения текста, музыка играет/ставится на паузу, **ритм-игра** (хелперы движка, старт
по кнопке, тап растит комбо, полоса не даёт горизонтальный скролл). Playwright сам
поднимает `python3 -m http.server 8123`.

Звук во всех тестах заглушён (`beforeEach` → `addInitScript` патчит
`HTMLMediaElement.prototype.play`: `muted=true`). Mute гасит только слышимость, не
состояние — поэтому спец-тесты «Музыка» проверяют `paused` без реального звука.

Chrome-проект использует **реальный установленный Chrome** (`channel: "chrome"` в
`playwright.config.js`) — скачанный Chromium не нужен (экономит ~536 МБ; кэш браузеров
~293 МБ = только WebKit). Требует, чтобы на машине был установлен Google Chrome.
Реальный Safari через Playwright недоступен (webkit — своя сборка, ближайший прокси к iOS).

Гочи тестов: подарок/буква постоянно анимируются → клик только с `{ force: true }`
(иначе Playwright бесконечно ждёт «стабильности»). Порог центра «Л» — 6px (разница
метрик шрифта WebKit/Chromium; реальные регрессии были 40px+). Фавикон задан data-URI
(эмодзи) — иначе Chrome ловит 404 на /favicon.ico (тест «нет ошибок консоли» это видит).

Тестовое окружение (`node_modules`, `package.json`, `playwright.config.js`, `tests/`)
**не попадает в деплой** — GitHub Pages отдаёт только статику.

## Персонализация

Всё в `js/config.js`: `name`, тексты `hero`, `finale.letter`/`finale.caption`, `music.url`,
а также тексты ритм-игры `finale.play`/`playHint`/`replayGame`/`gameEnd`.
Пустые поля → аккуратные плейсхолдеры, сайт не ломается.
