# CLAUDE.md — память проекта

Контекст для ИИ-ассистента и разработчика. Держи актуальным.

## Что это

Сайт-поздравление с днём рождения для **Кристины**. Личный подарок.
Деплой: **GitHub Pages**, репозиторий `happy-birthday-kristina`.

## Сцена (одна страница)

1. **Распаковка подарка** — коробка с бантом; тап (или тряска телефона) → конфетти → сайт.
2. **Hero** — «С днём рождения, Кристина» + подзаголовок, оркестрованный staggered-вход.
3. **Финал** — одна большая буква **«Л»** — это личное «люблю». Тап по букве → салют из
   разных эмодзи-сердечек (`confetti.shapeFromText`, красных больше розовых), вылетающих
   из самой буквы; маленькая кнопка внизу **«Ещё конфетти»** — дополнительный залп.

> Ранее были тортик / галерея / письмо — **удалены намеренно** по просьбе владельца. Не возвращать.

## Стек и принципы

- Чистые **HTML + CSS + JS, без сборки и без фреймворка**. Открывается браузером напрямую.
- **Без рантайм-CDN для логики:** `canvas-confetti` вшит локально (`js/confetti.js`).
  Единственная внешняя загрузка — Google Fonts (Playfair Display + Manrope).
- **iPhone-first (Safari)** — главный приоритет качества.

## Дизайн-направление

Одна тема — тёплый романтичный розовый. Принципы (по frontend-design):
доминирующий акцент `--accent` + золотой контрапункт `--gold`; атмосфера через
градиентный меш + тонкий SVG-шум + свечение за крупным текстом; связка шрифтов
**дисплейный курсив Playfair** (имя, «Л») + **Manrope** (текст, uppercase-трекинг в мелочах);
один срежиссированный вход со staggered-reveal вместо разрозненных микроанимаций.

Меняешь вид → правишь переменные в `:root` (`css/styles.css`). Больше тем нет — убраны намеренно.

## Гочи (не наступить повторно)

- **`hidden` + `display` в CSS:** любой явный `display` в авторских стилях перебивает
  HTML-атрибут `[hidden]`. Для скрываемых элементов держать правило `.x[hidden]{display:none}`
  (см. `.music-toggle`).
- **Автоплей аудио на iOS:** музыка стартует только внутри пользовательского жеста (тап по
  подарку/кнопке). Не пытаться автостартить без жеста.
- **`dvh`, а не `vh`:** иначе секции прыгают при скрытии адресной строки Safari.
- **`prefers-reduced-motion`:** уважать — частицы отключаются, reveal без сдвига.
- **Конфетти уважает reduced-motion** через `disableForReducedMotion: true`.
- **`background-clip: text` срезает свисающие части глифа:** завиток «Л» и italic-терминалы
  имени выходят за границы бокса, а фон красится только внутри бокса. Держать `padding` на
  `.finale-letter` и `.hero-name`, иначе чернила вне бокса не зальются (видно ровный срез).
- **Оптический центр «Л»:** глиф Playfair «Л» из-за правого бокового пробела смещён влево
  (~0.022em от центра advance-бокса). Выправлено `text-indent` на `.finale-letter`; при
  `text-align:center` indent двигает на половину, поэтому значение удвоено (~0.044em).
  **Меняешь шрифт/размер буквы → перепроверь оптический центр** (canvas `measureText`
  → `actualBoundingBoxLeft/Right`).
- **Зум на iOS отключён:** `viewport` c `user-scalable=no, maximum-scale=1` +
  `touch-action: manipulation` (снимает double-tap-зум). iOS игнорирует `user-scalable`,
  поэтому pinch дополнительно гасится в JS (`preventDefault` на `gesturestart/change/end`).
- **Overscroll-баунс:** фон-градиент продублирован на `html`, иначе по краям при «резинке»
  видны белые поля.

## Запуск и проверка

```bash
python3 -m http.server 8000   # http://localhost:8000 (посмотреть глазами)
npm test                       # E2E: WebKit(iPhone)+Chrome(Android), ~8с
```

**Тесты (Playwright) — основной способ верификации, быстрее ручного превью.**
`tests/e2e.spec.js` гоняет сайт в **реальном движке WebKit** (эмуляция iPhone, dpr 3) и
Chromium. Именно WebKit ловит iOS-специфику (горизонтальный скролл, швы фикс-слоёв),
которую Chromium-превью НЕ показывает — на этом мы не раз обжигались. Покрыто: нет
горизонтального переполнения, имя влезает и центрировано, «Л» оптически по центру,
частицы в границах вьюпорта, тап открывает только подарок, нет ошибок консоли, нет
выделения текста. Playwright сам поднимает `python3 -m http.server 8123`.

Chrome-проект использует **реальный установленный Chrome** (`channel: "chrome"` в
`playwright.config.js`) — скачанный Chromium не нужен (экономит ~536 МБ; кэш браузеров
~293 МБ = только WebKit). Требует, чтобы на машине был установлен Google Chrome.
Реальный Safari через Playwright недоступен (webkit — своя сборка, ближайший прокси к iOS).

Гочи тестов: подарок/буква постоянно анимируются → клик только с `{ force: true }`
(иначе Playwright бесконечно ждёт «стабильности»). Порог центра «Л» — 6px (разница
метрик шрифта WebKit/Chromium; реальные регрессии были 40px+). Фавикон задан data-URI
(эмодзи) — иначе Chrome ловит 404 на /favicon.ico (тест «нет ошибок консоли» это видит).

Тестовое окружение (`node_modules`, `package.json`, `playwright.config.js`, `tests/`)
**не попадает в деплой** — GitHub Pages отдаёт только статику.

## Персонализация

Всё в `js/config.js`: `name`, тексты `hero`, `finale.letter`/`finale.caption`, `music.url`.
Пустые поля → аккуратные плейсхолдеры, сайт не ломается.
