# Ритм-игра «в такт с Л» — дизайн

Дата: 2026-07-08. Проект: happy_birthday (сайт-поздравление Кристине).

## Что делаем

Добавляем простую ритм-игру под уже добавленную песню (Ghost — Mary on a Cross,
`assets/audio/mary-on-a-cross-loop.mp3`, ~72.7 c). Одна полоса, одна кнопка —
в стиле сердцебиения из Dark Pictures / караоке Yakuza. Игра — надстройка над
финальным экраном «Л»: буква остаётся, под ней появляется режим игры.

**Проиграть нельзя.** Это подарок, а не челлендж: попадания копят комбо и яркость,
промахи не карают.

## Флоу

1. Игрок доходит до финала — видит «Л» как сейчас (idle-анимация, тап → свободный
   всплеск сердечек — существующее поведение сохраняется).
2. Под «Л» — призыв (текст из конфига, напр. «Сыграй со мной в такт») и кнопка
   **«Играть»**.
3. Тап по «Играть» (пользовательский жест) → **музыка НЕ перезапускается**, игра
   подхватывает с текущей позиции трека (см. «Старт с текущей позиции» ниже) →
   короткий lead-in (полоса въезжает, `leadTime` = разгонная дорожка до первой ноты;
   опционально лёгкое «приготовься ✨», без жёсткого отсчёта 3-2-1 — музыка уже идёт).
   Идёт **режим игры**:
   - «Л» перестаёт idle-анимироваться и становится кнопкой-целью;
   - тап по «Л» (и по всей нижней зоне — удобнее на телефоне) = попытка попадания.
4. Ноты-импульсы едут справа к неподвижному кольцу-цели слева. Тап, когда нота в кольце:
   - **попадание** → кольцо вспыхивает + **всплеск обычного конфетти** из зоны «Л»
     (`boom()` — оптимизированнее эмодзи-сердечек), комбо++, сцена чуть ярче;
   - **промах** (нота прошла цель без тапа, или тап мимо окна) → нота гаснет, комбо
     мягко сбрасывается. Без штрафов, без game over.
5. **Конец — на конце трека** (см. «Конец игры» ниже): большой финальный залп конфетти
   + тёплая строка (из конфига) + кнопка «Ещё раз». **Музыка не останавливается** —
   продолжает крутиться в фоне без игры. «Ещё раз» перезапускает игру опять с текущей
   позиции; сцена возвращается к спокойной «Л».

### Старт с текущей позиции

Музыка автостартует при распаковке и зациклена (`loop`), так что к финалу трек уже
играет на произвольной фазе. Игру начинаем **с этой фазы, не перематывая** — этого
и просил владелец. Механика бесплатна: карта битов задана в координатах лупа
`[0, duration)`, а `audio.currentTime` у зациклённого `<audio>` крутится
0→duration→0, т.е. это и есть время внутри лупа. Движок берёт ближайшие предстоящие
доли от `now` — шва нет, музыка не прыгает. Поэтому жёсткий отсчёт 3-2-1 не нужен
(он для старта с тишины): роль разгона играет `leadTime`.

### Конец игры

Игра доигрывает до **конца текущего прохода трека** (граница лупа: `currentTime`
доходит до `duration` и оборачивается в 0), затем `onComplete`. Музыка при этом не
трогается — `<audio loop>` продолжает играть.

Краевой случай: если стартовать под самый конец лупа, игра вышла бы мгновенной.
Правило: **конец = первая граница лупа, до которой остаётся ≥ `MIN_PLAY` (~20 c)**.
Стартовал за 5 c до границы → доигрываешь до следующей (≈77 c игры). Одна строчка:
целевой конец = ближайшая граница `k·duration` (в абсолютном времени
воспроизведения) с зазором ≥ `MIN_PLAY`.

## Движок битов: запечённая карта

Синк «на глаз» плывёт, поэтому источник истины по времени — **само аудио**:
рантайм в `requestAnimationFrame` читает `audio.currentTime` и сверяет с массивом
времён долей. Так синк не уезжает даже при просадке кадров, и не нужен Web Audio
(капризный на iOS Safari).

Массив долей **запекается один раз оффлайн** из mp3 и хранится в
`js/rhythm-beatmap.js` (данные конкретной песни, отдельно от движка):

```js
window.RHYTHM_BEATMAP = {
  src: "assets/audio/mary-on-a-cross-loop.mp3",
  duration: 72.744,     // длина лупа, сек
  beats: [0.51, 0.93, 1.35, ...],  // времена долей в секундах от начала лупа
};
```

Извлечение (тулинг, п. «Тулинг» ниже): librosa `beat_track` → тайминги долей →
при необходимости прореживание/ручная правка под приятную плотность нот. Фолбэк,
если извлечение выйдет грязным: ровная сетка по BPM (~143) + сдвиг первой доли —
тот же формат массива, так что рантайм не меняется.

Аудио зациклено (`loop`), карта покрывает один луп; на переходе через `duration`
планировщик продолжает по модулю. Движок **не перематывает и не останавливает**
аудио — только читает `currentTime`; запуск/пауза музыки остаются на стороне сайта.

### Планирование нот

- `leadTime` (напр. 2.0 c) — за сколько до своей доли нота спавнится справа и едет к цели.
- В каждом кадре: `now = audio.currentTime` (время внутри лупа, 0…duration). Для каждой
  предстоящей доли `t` с `t - now <= leadTime` создаём ноту (если ещё не создана);
  позиция ноты по X интерполируется из `(t - now)` → от правого края к кольцу (0 в
  момент `t`). Через границу лупа предстоящие доли следующего прохода берём как
  `t + duration`, поэтому ноты у стыка едут без разрыва.
- **Окно попадания** `hitWindow` (напр. ±0.18 c): тап засчитывается ближайшей
  активной ноте, если `|t - now| <= hitWindow`. Иначе — промах-тап (ничего плохого,
  просто без конфетти; комбо не растёт).
- Нота, у которой `now - t > hitWindow` и по ней не тапнули — промах, гаснет.

### Отслеживание конца

Правило конца («первая граница лупа с зазором ≥ `MIN_PLAY`») требует **абсолютного**
времени игры, а `audio.currentTime` оборачивается на каждом лупе. Движок считает
обороты: замечает, когда `currentTime` резко падает (было близко к `duration`, стало
близко к 0) → `laps++`. Абсолютное время ≈ `laps * duration + currentTime`. При старте
запоминаем `startAbs`; целевой конец = наименьшее `k*duration ≥ startAbs + MIN_PLAY`.
Когда абсолютное время достигает цели → `stop()` + `onComplete` (аудио не трогаем).

## Портируемость (движок отдельно от сайта)

Требование владельца: логику потом можно вынуть в отдельную игру, но без
переусложнения. Значит:

- **`js/rhythm.js` — обобщённый движок**, один модуль-фабрика, без сборки/классовой
  иерархии. Ничего не знает про день рождения. API через options + колбэки:

  ```js
  var game = RhythmGame({
    audio: HTMLAudioElement,   // источник времени и звука
    mount: HTMLElement,        // куда рисовать полосу
    tapTargets: [el, ...],     // элементы-кнопки, тап по которым = попытка
    beats: [...], duration, leadTime, hitWindow,
    minPlay: 20,               // мин. длительность прохода до конца-на-границе
    onHit:   function (info) {},   // сайт вешает конфетти + комбо
    onMiss:  function (info) {},
    onComplete: function (stats) {},
    reducedMotion: bool,
  });
  game.start();  // подхват с текущей позиции audio + lead-in
  game.stop();   // очистка (rAF, ноты, слушатели); аудио НЕ трогает
  ```

- **`js/rhythm-beatmap.js`** — данные песни (см. выше), тоже отдельно.
- **Привязка к сайту — в `js/main.js`**: кнопка «Играть», пауза idle-анимации «Л»,
  колбэки (конфетти через `boom()`, счётчик комбо, финальная строка). Движок про это
  не знает. Так его можно скопировать в другой проект, передав свои `mount`/`audio`/
  `beats`/колбэки.
- Без переусложнения: без сборщика, без TS, без внешних зависимостей в рантайме —
  как и весь проект (чистые HTML+CSS+JS).

## iOS-гочи (не наступить)

- **Жест для звука:** музыка стартует/резюмится внутри тапа по «Играть». Не автостартить.
- **`pointerdown`, не `click`** для регистрации тапов — отзывчивее; `touch-action:
  manipulation` уже стоит.
- **Полоса не должна давать горизонтальный скролл:** `contain: layout paint` на
  контейнере полосы + ноты держим в границах по X (как у `.floaties`). WebKit-тест это ловит.
- **`dvh`, не `vh`** для любых новых высот.
- **`prefers-reduced-motion`:** конфетти уже само гаснет (`disableForReducedMotion`).
  Полосу оставляем работающей, но без тяжёлых частиц; передаём `reducedMotion` в движок,
  чтобы он мог упростить анимацию нот.
- **Аудио-часы:** `audio.currentTime` на iOS может иметь малую задержку относительно
  реального выхода звука — для подарка приемлемо; при заметном оффсете добавить
  калибровочную константу в beatmap.

## Тесты (Playwright, WebKit iPhone + Chrome)

Звук уже глушится в `beforeEach` (muted), но `currentTime` идёт → игру можно гонять.

- Кнопка «Играть» видна в финале; тап включает режим игры (появляется полоса).
- В режиме игры idle-анимация «Л» снята (класс/состояние).
- После старта ноты появляются в полосе (спавнятся при движении `currentTime`).
- Тап по «Л» в окне попадания → комбо растёт (проверяем состояние, не звук).
- Нет горизонтального переполнения при видимой полосе (ключевой iOS-тест).
- Нет ошибок консоли.
- Существующие тесты (имя, центр «Л», музыка, скролл) не сломаны.

Гоча тестов: элементы постоянно анимируются → клики с `{ force: true }` (как уже принято).
