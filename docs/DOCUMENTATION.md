# Документация проекта

Технический разбор сайта-поздравления. Для быстрого старта смотри
[README](../README.md); для контекста ассистента — [CLAUDE.md](../CLAUDE.md).

## 1. Обзор

Статический сайт из трёх сцен, без сборки. Загружается браузером напрямую, поэтому
деплой на GitHub Pages — это просто выкладка файлов.

```
happy_birthday/
├── index.html            разметка всех секций
├── css/styles.css        стили, палитра и шрифты (:root), анимации
├── js/
│   ├── config.js         ← ВСЯ персонализация
│   ├── main.js           логика сцен и интерактива
│   └── confetti.js       вшитый canvas-confetti (не трогать)
├── assets/audio/         mp3 для фоновой музыки (опционально)
├── docs/                 эта документация + дизайн-спека
├── CLAUDE.md             память проекта для ИИ-ассистента
└── README.md             быстрый старт и деплой
```

## 2. Поток сцен

### 2.1. Распаковка подарка (`#gift-screen`)
Полноэкранный оверлей с коробкой. Открывается по:
- **тапу** (клик по всему экрану), либо
- **тряске телефона** — слушатель `devicemotion`, порог суммарного ускорения > 32.

По открытию: класс `.opening` запускает анимацию «взрыва» коробки, стартует конфетти,
через 620 мс оверлей уходит (`.gone`), показывается `#site`, запускаются reveal и частицы.
Тап заодно служит пользовательским жестом для старта музыки (см. §5).

### 2.2. Hero (`#hero`)
Приветствие, имя, подзаголовок. **Оркестрованный вход**: элементы `.reveal` проявляются
со staggered-задержками (`--rd` в CSS: 0.05s → 0.22s → 0.44s → 0.7s).
Имя — курсивный Playfair с градиентной заливкой текста и радиальным свечением (`::after`).

### 2.3. Финал (`#finale`)
Одна крупная буква **«Л»** (из `config.finale.letter`) — личное «люблю».
Тап/Enter по букве → залп «сердечек» (крупные круглые частицы). Кнопка «Ещё раз сначала»
плавно скроллит наверх и повторяет конфетти.

## 3. Темизация и дизайн

Одна тема — тёплый романтичный розовый. **Вся палитра и шрифты — в `:root`**
(`css/styles.css`), это единственная точка правды:

- `--accent` / `--accent-deep` — доминирующий розовый акцент (градиенты текста, кнопки).
- `--gold` — тёплый контрапункт (бант, часть конфетти).
- `--confetti-colors` — список цветов конфетти (читается из CSS в JS).
- `--font-display` (Playfair Display) — заголовки, имя, «Л».
- `--font-body` (Manrope) — основной текст.

Атмосфера: многослойный радиальный градиентный меш на `body` + тонкий SVG-шум
(`body::before`, inline data-URI, без файлов) + свечение за крупным текстом.
Чтобы перекрасить сайт — меняешь переменные, разметку/логику трогать не нужно.

## 4. Reveal при скролле

`main.js` вешает `IntersectionObserver` (порог 0.18) на элементы `.reveal`, добавляя
класс `.in`. Первая секция раскрывается сразу после распаковки. Фолбэк: если
`IntersectionObserver` недоступен или включён `prefers-reduced-motion` — всё показывается сразу.

## 5. Музыка (опционально)

Задаётся `config.music.url`. Если пусто — кнопки нет (элемент `[hidden]`, а `.music-toggle[hidden]`
явно ставит `display:none`, чтобы `display:grid` не «показал» его). Если задано:
кнопка появляется, воспроизведение стартует по тапу (политика автоплея iOS), первый
тап по подарку тоже пытается запустить трек в рамках жеста.

## 6. Плавающие частицы

`main.js` генерирует 10–16 эмодзи-элементов (`🎈💛🩷✨🎉💫`) со случайными размером,
прозрачностью, длительностью, дрейфом и вращением (CSS-переменные + анимация `rise`).
Отключаются при `prefers-reduced-motion`.

## 7. iPhone-first: что важно

- `viewport-fit=cover` + `env(safe-area-inset-*)` — учёт чёлки и домашней полоски.
- `100dvh` вместо `vh` — секции не прыгают при скрытии адресной строки Safari.
- Тач-зоны ≥ 44–48px, никаких hover-only взаимодействий.
- Аудио — только по жесту.
- Полная поддержка `prefers-reduced-motion`.

## 8. Персонализация — чеклист

1. Открой `js/config.js`.
2. Поправь `name`, тексты `hero`, `finale.letter` (и `finale.caption` при желании).
3. Для музыки: положи файл в `assets/audio/`, впиши путь в `music.url`.
4. Проверь локально (`python3 -m http.server`), затем задеплой (см. README / скилл `/deploy`).

## 9. Ограничения (вне scope)

Нет бэкенда, RSVP, сбора данных, аналитики, сборки/бандлера и рантайм-CDN для логики.
Сознательный минимализм ради надёжности и простоты деплоя.
