# Документация проекта

## 1. Обзор

Статический сайт из двух сцен (после распаковки), без сборки. Загружается браузером
напрямую, поэтому деплой на GitHub Pages — это просто выкладка файлов.

```

├── manifest.json           PWA-манифест (имя, иконки, standalone, цвета) — см. §12
├── sw.js                   service worker (офлайн-кэш app-shell + Google Fonts) — см. §12
├── css/styles.css          стили, палитра и шрифты (:root), фон, анимации
├── js/
│   ├── config.js           ← ВСЯ персонализация
```

## 2. Поток сцен

### 2.1. Распаковка подарка (`#gift-screen`)
Полноэкранный оверлей с коробкой-подарком. Открывается по:
- **тапу по самому подарку** (`#gift`, не по всему экрану — как и написано в подсказке),
- либо **тряске телефона** — слушатель `devicemotion`, порог суммарного ускорения > 32.

По открытию: класс `.opening` запускает анимацию «взрыва» коробки, стартует конфетти,
через 620 мс оверлей уходит (`.gone`), показывается `#site`, вызывается `fitName()` (§2.2),
запускаются reveal и частицы. Тап по подарку заодно служит пользовательским жестом для
старта музыки (§5).

### 2.2. Hero (`#hero`)
Приветствие, имя, подзаголовок. **Оркестрованный вход**: элементы `.reveal` проявляются
со staggered-задержками (`--rd` в CSS: 0.05s → 0.22s → 0.44s → 0.7s).
Имя — курсивный Playfair с мягкой градиентной заливкой текста и радиальным свечением (`::after`).

**Авто-подгон имени по ширине.** Размер задан `clamp(2.6rem, 15.5vw, 6.5rem)` (масштаб по
ширине экрана), а `fitName()` в `main.js` дополнительно уменьшает кегль, если имя всё же
шире ~92% экрана (длинное имя / узкий экран). Вызывается при раскрытии и на resize.

### 2.3. Финал (`#finale`)
Одна крупная буква **«Л»** (из `config.finale.letter`) — личное «люблю».
Тап/Enter по букве → залп из **разных эмодзи-сердечек**, вылетающих из самой буквы:
`heartsBurst()` создаёт формы через `confetti.shapeFromText` (набор `❤️ ❤️ ❤️‍🔥 ❤️‍🔥 🩷 💕 💖 💘`
— красные повторены, чтобы их было больше), origin берётся из `getBoundingClientRect()` буквы.
Маленькая второстепенная кнопка внизу **«Ещё конфетти»** даёт дополнительный залп (`celebrate()`),
никуда не скроллит.

Под «Л» — **ритм-игра** (кнопка «Играть в такт»): жмёшь в такт уже играющей песне.
Полный разбор — §13. Про центрирование буквы — см. §8.

## 3. Фон, палитра и дизайн

Одна тема — тёплый романтичный розовый. **Вся палитра и шрифты — в `:root`**
(`css/styles.css`), единственная точка правды:

- `--accent` / `--accent-deep` — доминирующий розовый акцент (градиенты текста, кнопки).
- `--gold` — тёплый контрапункт (бант подарка).
- `--confetti-colors` — цвета конфетти для `celebrate()` (читаются из CSS в JS).
- `--font-display` (Playfair Display) — заголовки, имя, «Л».
- `--font-body` (Manrope) — основной текст.

**Фон-атмосфера вынесена в отдельный фиксированный слой `<div class="bg">`** (z-index −1):
многослойный розовый радиальный меш + тонкий SVG-шум (`.bg::after`, inline data-URI, без файлов).
Это НЕ `background-attachment: fixed` — то свойство даёт швы/артефакты на iOS Safari (ранняя версия
на нём и ломалась). `body` при этом **прозрачный** (`background: transparent`), иначе его непрозрачная
заливка перекрыла бы слой `.bg` (баг «градиенты исчезли под заливкой»). Базовый цвет живёт на `.bg`,
а на `html` продублирован градиент — чтобы при overscroll-«резинке» по краям не мелькали белые поля.

Чтобы перекрасить сайт — меняешь переменные `:root` и градиенты `.bg`, разметку/логику не трогаешь.

## 4. Reveal при скролле

`main.js` вешает `IntersectionObserver` (порог 0.18) на элементы `.reveal`, добавляя
класс `.in`. Первая секция раскрывается сразу после распаковки. Фолбэк: если
`IntersectionObserver` недоступен или включён `prefers-reduced-motion` — всё показывается сразу.

## 5. Музыка (опционально)

Задаётся `config.music.url`. Если пусто — кнопки нет (элемент `[hidden]`, а `.music-toggle[hidden]`
явно ставит `display:none`, чтобы `display:grid` не «показал» его). Если задано:
кнопка появляется, воспроизведение стартует по тапу (политика автоплея iOS), первый
тап по подарку тоже пытается запустить трек в рамках жеста. Кнопка ставит на паузу/включает.

## 6. Плавающие частицы

`main.js` генерирует 10–16 эмодзи-элементов (`🎈❤️🩷✨🎉💫`) со случайными размером,
прозрачностью, длительностью, дрейфом и вращением (CSS-переменные + анимация `rise`).
Отключаются при `prefers-reduced-motion`.

Позиции по X держатся **внутри вьюпорта** (левый край 3–82vw, дрейф ±4vw), а слой `.floaties`
обрезается через `overflow: clip` + `contain: layout paint size`. Причина: на iOS Safari
`overflow:hidden` фиксированного слоя **не обрезает** промоутнутые (transform) дочерние
элементы — они вылезали за экран и давали горизонтальный скролл (в Chromium этого бага нет).

## 7. iPhone-first: что важно

- `viewport-fit=cover` + `env(safe-area-inset-*)` — учёт чёлки и домашней полоски.
- `100dvh` вместо `vh` — секции не прыгают при скрытии адресной строки Safari.
- Тач-зоны ≥ 44–48px, никаких hover-only взаимодействий.
- Аудио — только по жесту.
- Полная поддержка `prefers-reduced-motion`.
- Выделение текста отключено (`user-select: none` на `body`), чтобы двойной тап по «Л» не выделял.
- **Зум отключён:** `viewport` c `user-scalable=no, maximum-scale=1` + `touch-action: manipulation`
  (снимает double-tap-зум). Safari на iOS игнорирует `user-scalable`, поэтому pinch-жесты
  дополнительно гасятся в `main.js` (`preventDefault` на `gesturestart/gesturechange/gestureend`).
- **Overscroll-баунс:** градиент на `html` (в дополнение к `.bg`) — иначе по краям видны белые поля.
- **Горизонтальный скролл:** `overflow-x: hidden` на `html`+`body`; частицы держатся в границах (§6).

## 8. Крупная буква «Л»: клиппинг и оптический центр

Финальная «Л» набрана дисплейным Playfair и залита градиентом через `background-clip: text`.
Отсюда две тонкости, которые легко сломать:

1. **Клиппинг свисающих частей.** `background-clip: text` красит текст только в пределах
   бокса элемента. У Playfair «Л» левый завиток-терминал (и italic-терминалы в имени)
   выходят за advance-бокс — без запаса они не заливаются и видны **ровным срезом**.
   Лечится внутренними отступами: `.finale-letter { padding: 0.05em 0.22em }`
   (и аналогично `.hero-name`). Меняешь размер/шрифт — проверь, что завиток целиком в заливке.

2. **Оптический центр.** Даже когда advance-бокс математически по центру (проверено:
   `getBoundingClientRect` → offset 0), сам глиф «Л» выглядит смещённым влево — из-за
   правого бокового пробела чернила смещены от центра бокса на ~`0.022em`. Компенсируется
   `text-indent` на `.finale-letter`. Нюанс: при `text-align: center` `text-indent` сдвигает
   содержимое на **половину** своего значения, поэтому в CSS стоит удвоенное `~0.044em`
   (итоговый оптический offset ≈ 0).

   **Как перепроверить при смене шрифта/размера буквы** (canvas, без глаз):
   ```js
   var cs = getComputedStyle(el), fs = parseFloat(cs.fontSize);
   var c = document.createElement('canvas').getContext('2d');
   c.font = cs.fontStyle+' '+cs.fontWeight+' '+fs+'px '+cs.fontFamily;
   var m = c.measureText('Л');
   var shiftPx = m.width/2 - (-m.actualBoundingBoxLeft + m.actualBoundingBoxRight)/2;
   // нужный text-indent ≈ 2 * shiftPx (из-за text-align:center)
   ```

## 9. Тесты (Playwright) — основной способ проверки

```bash
npm test                 # WebKit(iPhone) + реальный Chrome, ~25с
npm run test:report      # HTML-отчёт последнего прогона
```

`tests/e2e.spec.js` гоняет сайт в **реальном движке WebKit** (эмуляция iPhone, dpr 3) и
в **реальном установленном Chrome** (`channel: "chrome"`, без скачанного Chromium).
Именно WebKit ловит iOS-специфику (горизонтальный скролл, швы фикс-слоёв), которую
Chromium-превью НЕ показывает — на этом не раз обжигались. Playwright сам поднимает
`python3 -m http.server 8123`.

Покрыто: нет горизонтального переполнения; имя влезает и центрировано; «Л» оптически
по центру; частицы в границах вьюпорта; открывается только по тапу на подарок; нет ошибок
консоли; нет выделения текста; музыка играет и ставится на паузу; **PWA** — манифест
валиден и пути относительны, иконки отдаются без 404, service worker регистрируется;
**ритм-игра** (§13) — чистые хелперы движка (`endTarget/noteX/nextLaps`), старт по кнопке
(полоса видна, idle-анимация «Л» снята, ноты появляются), тап растит комбо, полоса не даёт
горизонтальный скролл.

Особенности:
- **Клик по анимированным элементам — только `{ force: true }`** (подарок/буква постоянно
  анимируются, иначе Playwright бесконечно ждёт «стабильности»).
- **Звук заглушён во всех тестах**: `beforeEach` → `addInitScript` патчит
  `HTMLMediaElement.prototype.play` (`muted=true`). Mute гасит только слышимость, не состояние
  `paused`, поэтому спец-тесты «Музыка» всё равно проверяют, что трек играет.
- **Порог оптического центра «Л» — 6px**: метрики шрифта WebKit/Chromium чуть расходятся;
  реальные регрессии центра были 40px+.
- **Фавикон — data-URI (эмодзи 🎂)**: иначе Chrome ловит 404 на `/favicon.ico`, и тест
  «нет ошибок консоли» краснеет.
- Реального **Safari** Playwright не запускает (webkit — своя сборка, ближайший прокси к iOS);
  Chrome берётся системный.

## 11. Ограничения (вне scope)

Нет бэкенда, RSVP, сбора данных, аналитики, сборки/бандлера и рантайм-CDN для логики.

## 12. PWA (установка на «Домой» + офлайн)

Сайт устанавливается иконкой-приложением и работает офлайн. Всё — статикой, без сборки.

**Файлы:**
- `manifest.json` — `standalone`, `portrait`, `background_color:#ffe1ec` (светло-розовый
  сплэш в тон сайта), `theme_color:#ff7eb3`; иконки 192/512 (`any`) + maskable-512.
  **`start_url` и `scope` — относительные (`"./"`)**, как и `src` иконок.
- `sw.js` — service worker. `install`: предкэш app-shell (html/css/js/аудио/иконки/манифест)
  относительными путями от `self.registration.scope`. `fetch`: **cache-first** для своего
  origin, **stale-while-revalidate** для Google Fonts (кэш `birthday-fonts-v1`). `activate`:
  чистит кэши с чужой версией. Имя кэша версионируется — `birthday-v1`.
- В `index.html`: `<link rel="manifest">`, `apple-touch-icon`, apple-мета
  (`apple-mobile-web-app-capable/-status-bar-style/-title`) и инлайн-регистрация SW
  (только если `serviceWorker in navigator` и протокол не `file:`).

**Иконки** — `assets/icons/*.png`, из ❤️ на розовом градиенте сайта. Генерятся разово
скриптом `scripts/gen-icons.mjs` (`npm run icons`) через Playwright/Chrome (`channel:"chrome"`,
скачанный Chromium в проекте отсутствует). PNG **коммитятся**; в рантайме ничего не рендерится.
Путь-сердечко — симметричный Material «favorite» (несимметричный самодельный давал «кривое»).

**Гочи (тоже в CLAUDE.md):**
- **Относительные пути везде** — из-за подпапки GitHub Pages (`/happy-birthday-kristina/`).
  Абсолютные `/...` смотрят в корень домена и ломают офлайн/установку на Pages.
- **iOS ≠ манифест:** установку на «Домой» на iPhone держат apple-мета + `apple-touch-icon`,
  не только `manifest.json`.
- **Версию кэша `birthday-vN` поднимать при смене закэшированных ассетов** — иначе у уже
  установивших останется старьё (cache-first не идёт в сеть за свежим).
- **Меняешь тему/палитру → пере-генерируй иконки** (`npm run icons`): цвета зашиты в
  `gen-icons.mjs` (держать в синхроне с `:root`).

## 13. Ритм-игра под «Л»

Простая ритм-игра на финальном экране: одна горизонтальная полоса, одна кнопка (сама «Л»),
в стиле сердцебиения из Dark Pictures. **Проиграть нельзя** — попадания копят комбо, промахи
не карают.

### Флоу
Под «Л» — призыв (`config.finale.playHint`) и кнопка **«Играть в такт»** (`config.finale.play`).
Тап → **музыка не перематывается**, игра подхватывает трек с текущей позиции; `#finale` получает
класс `.playing` (idle-анимация «Л» снята, «Л» становится кнопкой-целью); показывается полоса
`#rhythm`. **Полоса — НАД буквой** (в разметке `#rhythm` идёт до `#finale-letter`): иначе рука при
тапе по «Л» её закрывает. Ноты едут справа к кольцу-цели слева; тап по «Л» (или по полосе) в момент
совпадения — попадание: кольцо вспыхивает, **«Л» коротко «попает»** (`letter-pop`), из зоны «Л»
летит **обычное конфетти** (`boom()` — оптимизированнее эмодзи-сердечек), растёт комбо `×N`. Окно
попадания симметричное и щедрое (`hitWindow: 0.30`, `leadTime: 2.4` из `main.js`) — легко попадать
и до, и после доли; нота при этом **проезжает сквозь кольцо** и уезжает левее (см. `noteX` ниже),
а не застревает в центре. По концу трека —
большой залп (`celebrate()`), строка `config.finale.gameEnd` и кнопка **«Ещё раз»**
(`config.finale.replayGame`), музыка продолжает играть в фоне.

### Движок (`js/rhythm.js`) — портируемый
Обобщённая фабрика `window.RhythmGame(opts) -> { start, stop }`, ничего не знает про день рождения;
вся привязка к сайту — в `main.js` через колбэки `onHit/onMiss/onComplete`. Ключевое:
- **Источник времени — аудио.** Каждый кадр читается `audio.currentTime` (не свои таймеры) —
  синк не плывёт при просадке кадров, Web Audio не нужен. Движок аудио **не трогает** (не
  перематывает, не останавливает).
- **Планирование нот** от `now = audio.currentTime`: доля с `t - now ≤ leadTime` спавнит
  ноту; позиция по X = `noteX(t-now, leadTime)`. `noteX` **не ограничен сверху**: при `dt < 0`
  (доля прошла, но ещё в окне) нота проезжает сквозь кольцо-цель и уезжает левее, а не залипает
  в центре. Окно попадания `hitWindow` симметрично (тап засчитывается при `|t-now| ≤ hitWindow`).
- **Конец — на границе трека.** `audio.currentTime` у зациклённого `<audio>` крутится
  `0…duration`; движок считает обороты (`nextLaps`, гистерезис 0.5 c) → абсолютное время.
  Конец = `endTarget(startAbs, duration, minPlay)` — ближайшая граница `k·duration` с зазором
  ≥ `minPlay` (10 c), чтобы старт под конец лупа не давал мгновенную игру.
- Чистые хелперы вынесены на `RhythmGame.util` (`endTarget/noteX/nextLaps`) и покрыты тестами.

### Карта битов (`js/rhythm-beatmap.js`) — данные, отдельно от движка
`window.RHYTHM_BEATMAP = { src, duration, beats:[…] }` — времена долей (сек от начала лупа).
**Сгенерено**, руками не править. Текущая карта — **режим акцентов** (`--accent`): не ровный
метроном, а ключевые атаки песни (онсеты, отобранные по силе) → неравномерный, музыкальный ритм
с кластерами и паузами. Дефолт — 69 нот (`--min-gap 0.55 --top 0.7`).

**Проще всего менять плотность нот — обёртка `tools/beatmap/regen.sh`:**
```bash
tools/beatmap/regen.sh              # дефолт (69 нот)
tools/beatmap/regen.sh 0.45 0.85    # больше нот (плотнее)
tools/beatmap/regen.sh 0.80 0.50    # меньше нот (больше пауз)
# аргументы: MIN_GAP (больше = меньше нот), TOP — доля сильнейших 0..1 (больше = больше нот)
```
Под капотом (можно и напрямую):
```bash
tools/beatmap/.venv/bin/python tools/beatmap/extract.py \
  assets/audio/<файл>.mp3 --src assets/audio/<файл>.mp3 \
  --out js/rhythm-beatmap.js --accent --min-gap 0.55 --top 0.7
```
Три способа добычи (все дают один формат — массив времён, рантайм не меняется):
- **`--accent`** (текущий) — онсеты по силе (`onset_strength`/`onset_detect`), музыкально/неравномерно;
- **по умолчанию** — `beat_track` (ровная сетка долей) + прореживание `--min-gap`;
- **`--bpm <темп> --offset <сек>`** — синтетическая ровная сетка (фолбэк без анализа).

venv (`tools/beatmap/.venv/`) — в `.gitignore`, сам `extract.py` коммитится.

### Стили
Полоса `.rhythm-lane` — `contain: layout paint` + `overflow:hidden`, чтобы ноты с `transform`
не давали горизонтальный скролл (та же iOS-гоча, что у `.floaties`, §6). Кольцо-цель `.rhythm-target`
на `left:12%` — **синхронно с `targetPct:12` в `js/rhythm.js`** (меняешь одно — меняй другое).
Цвета — переменные `--rg-lane/--rg-target/--rg-note` в `:root`.
